Tal y como esta ahora no se puede hacer un reparto de las notas que se 
generan a partir de una progresion en diferentes instrumentos con 
continuidad armonica.
Se me ha ocurrido una forma pero es un poco gena ( -ro ). Para ello necesito 
que se pueda especificar una semilla por el usuario. Una semilla simplemente 
puede ser un entero. Ahora imaginemos que tenemos varios instrumentos con 
una misma progresion. Si la semilla es la misma para todos y el numero de 
notas tambien entonces las notas que sacarian serian las mismas. Ahora 
tendriamos que aplicar un patron ritmico distinto a cada instrumento. Dicho 
patron ritmico simplemente tendria que especificar la voz que va a ejecutar 
ese instrumento.
Por eso quiero que en interfaz haya dos opciones de aplicar semilla: una que 
sea un cuadro de texto en el que se pueda escribir una semilla como un 
entero; otra sea un entero aleatorio (una semilla aleatoria, cosa que en si 
misma creo que es una contradiccion, pero bueno) y ya se encargara haskell 
se buscar dicho entero, por ejemplo, con la hora actual del sistema.
Creo que no parece mala idea y daria mucho juego, podriamos hacer un 
quinteto de viento o cosas de esas.
