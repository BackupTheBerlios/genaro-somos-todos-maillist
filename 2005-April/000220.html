>>La idea es usar un estándar para unificar el uso de numeros aleatorios en 
>>el codigo de HASKELL. Se basa en:
>>
>>-En BiblioGenaro.hs se ha definido
>>
>>resolucionRandom :: Int
>>resolucionRandom = 1000
>>
>>todos los numeros aleatorios se generarán entre 1 y resolucionRandom . De 
>>esta forma no tenemos numero bailando entre el codigo y podemos tener mas 
>>o menos aleatoriedad cambiando un numero. Podemos suponer que cualquier 
>>numero aleatorio/resolucionRandom pertenece a [0,1], y q multiplicado por 
>>otro aplica un porcentaje
>>
>>- Los numeros aleatorios de los procesos pueden generarse con  aleat <- 
>>listaInfNumsAleatoriosIO 1 resolucionRandom, que devuelve una lista 
>>infinita de enteros aleatorios entre 1 y resolucionRandom. Si las 
>>funciones que usan numeros aleatorios tienen de primer parametro una lista 
>>de enteros aleatorios y devuelven una paraja formada por el elemento que 
>>se quiera (lo que se quiere que devuelva la funcion) y los numeros 
>>aleatorios de la lista que no se hayan usado, tenemos una manera facil de 
>>hacer el paso de numeros aleatorios entre las funciones
>>
>>- Las funciones
>>
>>{-
>>Dada una lista de parejas (termino, peso), donde los pesos son naturales, 
>>devuelve en Elem un elemento
>>(es decir, primer componente de una de las parejas que forman la lista de 
>>entrada) elegido al azar entre
>>de los de la lista asignando a cada elemento/pareja una probabilidad de 
>>ser elegida igual a (peso/sumaPesos)
>>, donde suma pesos es la suma de los pesos de todos los elementos de la 
>>lista. Para ello se le debe suministrar un
>>numero aleatorio entre 1 y resolucionRandom. Se devuelve el elemento 
>>elegido y su posicion
>>-dameElemAleatListaPesos aleat listaParejas
>>-}
>>dameElemAleatListaPesos :: Int -> [(a, Int)] -> (a, Int)
>>

>>
>>{-
>>como dameElemAleatListaPesos pero con pesos de tipo Float
>>-}
