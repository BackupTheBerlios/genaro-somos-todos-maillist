Para la documentacion puede ser necesario hacer diagramas de bloques o 
clases para explicar como funcionan los algoritmos. Si el codigo se entiende 
solo es casi mas facil ponerlo directamente, si no a lo mejor basta con 
escribir pseudo-codigo, pero de todas formas es probable q acabemos metiendo 
diagramas q se integran muy facilmente con LaTeX pinchandolos como png o 
jpg. Este mensaje es solo para sugeriros que useis el viejo Violet 
(http://www.horstmann.com/violet/#download) o si usais Linux o esta para 
Cygwin el editor dia, q es cojonudo. Tiene UML y un monton de dibujines mas 
y es muy flexible y facil de usar, yo la verdad es q he flipado, pero es q 
soy muy impresionable
